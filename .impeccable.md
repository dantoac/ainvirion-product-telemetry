## Design Context

### Users
Developers, CTOs, and small technical teams who need self-hosted product analytics. They value data sovereignty, minimal infrastructure footprint, and tools that get out of the way. They're evaluating this while reading docs, comparing alternatives, and deciding whether to spin up a container this afternoon.

### Brand Personality
**Direct, reliable, pragmatic.** AInvirion builds tools for people who ship. No hand-waving, no marketing theater — show what it does, how it works, and how to run it.

### Emotional Goals
- **Confidence**: "This team knows what they're doing"
- **Control**: "I own every piece of this"
- **Simplicity**: "I can run this in an afternoon"

### Aesthetic Direction
- **Tone**: Technical, grounded, terminal-adjacent — but polished enough to feel like a real product, not a side project
- **Theme**: Dark default with a cool celeste-tinted light mode. The light theme should feel like a bright IDE theme — technical, clean, slightly blue — not a marketing site
- **Accent**: Phosphor lime (oklch hue ~115-130) — sharp, confident, not decorative
- **Type**: Hanken Grotesk (display/body) + JetBrains Mono (code/labels) — already established, keep it
- **Anti-references**: Vercel (too polished/gradient-heavy), generic AI startup aesthetic (purple-blue gradients, glow effects, dark-mode-with-neon)
- **References**: PostHog's directness (but lighter footprint), Linear's restraint, Tailscale's honesty

### Design Principles
1. **Show, don't decorate** — Every visual element should communicate something. If it doesn't inform, remove it.
2. **Terminal confidence** — The aesthetic says "I run in your terminal." Code blocks, monospace labels, numbered lists — these aren't decoration, they're the language of the audience.
3. **Earned color** — Color is semantic, not decorative. Accent marks action. Status colors mark state. Background tints create hierarchy. Nothing else gets color.
4. **Honest surfaces** — No frosted glass, no glow, no blur effects pretending to be depth. Flat surfaces with subtle tints and real shadows.
5. **Two-minute rule** — If someone can't understand what this product does and how to run it within two minutes of landing, the design failed.
