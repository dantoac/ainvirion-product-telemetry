FAQ

Things you actually want to know.

  1. 01 Can I really run this on my own box? +
    Yes. There's a Dockerfile, a docker-compose.yml, and a DigitalOcean App Platform spec. Bring Postgres 16 and a small VM and you're running.
  2. 02 Why not just use PostHog or Mixpanel? +
    Because sometimes you can't ship user identifiers to a third party — and "self-hosted PostHog" has a footprint we couldn't justify for a small team. Product Telemetry is FastAPI + Postgres. That's it.
  3. 03 What's the SDK story? +
    A single REST endpoint with an API key header. Wrap it in 12 lines of whatever language you're in. We ship thin TS and Python helpers, no obligation to use them.
  4. 04 Is the source code open? +
    Source-available under NDA, not OSS. You can read, run, and modify it for your own use. Redistribution is restricted. The license file in the repo is authoritative.
  5. 05 How do I report a bug or get help? +
    Open an issue on the AInvirion support repo on GitHub. That's the one and only support channel — no email, no Discord, no Slack workspace to lose track of.
  6. 06 What happens to my data on the managed plan? +
    It lives in a dedicated Postgres instance on AInvirion infrastructure (Seattle or Santiago region). Daily encrypted backups, 30-day retention, GDPR data subject endpoints exposed.
